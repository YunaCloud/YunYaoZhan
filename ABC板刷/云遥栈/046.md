## C - 原神和鸣潮 (AtCoDeer and Election Report)

### Description

yunayu 正在电视上看一场选举结果的快报。此次选举有两款候选游戏：原神和鸣潮。报道中仅展示了两款游戏当前得票数的最简比例，并未显示实际票数。

yunayu 一共看了 $n$ 次报告，当她第 $i$ 次查看时，两款游戏得票的比例是 $T_i:A_i$。已知她在第一次查看报告时，两款候选游戏都至少已获得一票。

请找出她在第 $n$ 次查看报告时，两款游戏得票总数的最小可能值。可以假设每款候选游戏的得票数永远不会减少（即只会增加或保持不变）。

### Solution

可以想到一个很正确的策略，就是每一轮结束时两人的票数一定分别是她每一轮看到的比例的若干倍，只要我们每一轮都取最小的倍数，那么最终得到的票数也一定是最小的。

当前票数与当前比例的大小关系可能有很多种，都大于，都小于，或是一大一小，之类的，然而稍加分析可得我们一定要取到的是最小的 $k$，使得比例翻 $k$ 倍后分别都要不小于当前票数。那么本轮结束时最小的票数就一定是这两个翻倍后的比例了。

那么就是怎么求每一轮的 $k$ 的问题了。二分或者倍增都很好解决，然而一定要用 $log$ 吗……

是的，每轮变化可以 $O(1)$ 的解决……（*写的时候完全没有去想，已心碎*） 

我们可以分开看，当我们希望一个数翻若干倍之后不小于一个数的时候，其实可以直接除然后上取整。那么两个数的话就是分别求出最少需要翻的倍数然后取 $max$ 即可。

不如再推广一下，那么对于一群数而言，我们对所有的结果取 $max$ 同样可以实现目的。

其实这里的 $max$ 还可以看作是一种取交集的操作，换句话说，只有当所有组成部分都满足条件时，整体才合法。

虽然不难，但是受益匪浅啊，学到了。给我的感觉和之前一个 $1200$ 很类似，同样是本来可以线性处理，但是我套了二分。当时得知后我也琢磨了很久。虽然是很不起眼的东西，但或许可以在以后的某道题上打开思路。

### Code

```
void solve() {
    cin>>n; for(int i=1; i<=n; i++) cin>>u[i]>>v[i];
    int yu=0, na=0;
    for(int i=1; i<=n; i++) {
        int k=max(1LL, max((yu+u[i]-1)/u[i], (na+v[i]-1)/v[i]));
        yu=k*u[i]; na=k*v[i];
    }
    cout<<yu+na<<'\n';
}
```

## D - 石头滚木布 (AtCoDeer and Rock-Paper)

### Description

优优和娜娜正在玩一个游戏。该游戏共进行 $n$ 回合。在每一回合中，两位玩家都需要像“石头剪刀布”那样出“石头”或“布”中的其中一种手势，但必须满足以下条件：

- 每一回合结束后，（玩家出布的次数）≦（玩家出石头的次数）。

每位玩家的得分计算方式为：（玩家获胜的回合数）减去（玩家输掉的回合数）。

> 每一回合的胜负由石头剪刀布的规则决定：
> - 如果一方出石头，另一方出布，则出布的一方获胜，出石头的一方输。
> - 如果双方出相同的手势，则为平局，双方均不得分（既不算赢也不算输）

游戏开始前，优优凭借她的超能力，已经预知了娜娜在接下来 $n$ 回合中每一回合会出的手势。请规划优优在每一回合应该出什么手势，以使她的得分最大化。输出这个得分。


### Solution

#### 我的思路

贪心中非常经典的一个 trick，就是考虑延迟策略能否带来额外收益，而这个 trick 很常见的一个标志就是不同操作带来的收益或亏损相等。

##### 感性理解

对于本题，感性的理解就是：如果我这回合本来可以出布，但我留着不出，本回合先出石头，后面的某个回合再出布，并不能带来额外的收益。

##### 证明

因为无论在什么情况下，出布和出石头的收益差都是 1，并且多出一次石头也只能增加一次出布的机会。本回合 -1，后面的某个回合 +1，并不会使得总分增加。

所以延迟出布不可能带来额外收益，而且反而可能会带来别的问题。比如我多屯了一些出布的次数，但是回合数已经耗尽了，就造成了资源的浪费。

所以最终的策略就是有布就出。

#### 题解的思路

题解就相对比较小巧思了，更像是从全局的把握。

假设优优每次都出石头，那么她的得分就是娜娜出布次数的相反数。而无论优优在哪一回合将出石头改为出布，她的得分都会恰好 +1。

那么最终就是尽量多出布了，次数也就是回合数除以 2 向下去整。

### Code

```
void solve() {
    string s; cin>>s; int cntg=0, cntp=0, __=0;
    for(auto &c: s) {
        if(cntg>cntp) {
            if(c=='g') __++; cntp++;
        }
        else {
            if(c=='p') __--; cntg++;
        }
    }
    cout<<__<<'\n';
}
```
