## D - Snuke's Coloring

### Description

yunayu 有一个 $h \cdot w$ 的网格。最初，所有的格子都是白色的。

yunayu 将其中的 $n$ 个格子涂成了粉色。第 $i$ 个被涂粉的格子位于第 $a$ 行、第 $b$ 列。

对于每个整数 $j\ (0 \le j \le 9)$，yunayu 想知道，整个网格中有多少个 $3 \cdot 3$ 大小的子矩形恰好包含 $j$ 个粉色格子？

> - $3 \le h \le 10^9$
> - $3 \le w \le 10^9$
> - $0 \le n \le min(10^5, h \cdot w)$
> - $1 \le a_i \le h$
> - $1 \le b_i \le w$
> - $(a_i, b_i) \ne (a_j, b_j)\ (i \ne j)$

### Solution

首先这个网格非常的大，意味着我们不能朴素地计数，但是注意到点的个数是非常有限的。

题目要求输出含有 0 个粉色格子的子矩形数，这个时候考虑白色格子一定是不好考虑的，可以考虑容斥。换句话说，这个值就等于 $3 \cdot 3$ 格子总数与所有含有粉色格子的数量总和的差。

那么对于 $j\ (1 \le j \le 9)$，我们可以这样想，一个涂成粉色的格子只会给特定的几个 $3 \cdot 3$ 格子带来粉色格子数 $+1$ 的效果，map 很好维护。

最后考虑每一个被影响的格子所含有的粉色格子数，然后就写完了！

总结一下就是 map + 算贡献。

### Code

```
void solve() {
    cin>>h>>w>>n; map<pair<int, int>, int> m;
    for(int i=1; i<=n; i++) {
        int u, v; cin>>u>>v;
        for(int x=u-2; x<=u; x++) {
            for(int y=v-2; y<=v; y++) {
                if(x>=1&&y>=1&&x+2<=h&&y+2<=w) m[{x, y}]++;
            }
        }
    }
    ans[0]=(h-2)*(w-2);
    for(auto &[p, cnt]: m) {
        ans[cnt]++; ans[0]--;
    }
    for(int i=0; i<=9; i++) cout<<ans[i]<<'\n';
}
```
